# 🗄️ PixelStore Database Integration System

## 📖 Project Overview

PixelStore Database Integration System is a full-stack web application developed to demonstrate practical implementation of database-driven software architecture. The project focuses on integrating a relational database with a backend application to efficiently store, manage, retrieve, and process product, order, and review information.

The system utilizes SQLite as the primary database management system and Flask as the backend framework, enabling seamless communication between the frontend interface and persistent database storage.

This project was developed as part of a Full Stack Development Internship to gain hands-on experience in database design, SQL operations, backend integration, and data management.

---

## 🎯 Objectives

* Design and implement a relational database schema
* Establish database connectivity using Python and SQLite
* Perform Create, Read, Update, and Delete (CRUD) operations
* Store application data persistently
* Retrieve and process information efficiently
* Integrate frontend and backend components with a database
* Build a scalable and maintainable data management system

---

## 🗃️ Database Design

The database was designed using a relational structure consisting of multiple interconnected entities.

### Products Table

Stores product information including:

* Product ID
* Product Name
* Price
* Category
* Product Image
* Product Description

### Orders Table

Stores order-related information including:

* Order ID
* Total Order Value

### Reviews Table

Stores customer feedback including:

* Review ID
* Product ID
* Review Content

The schema follows structured database design principles to ensure data consistency, organization, and efficient retrieval.
